### 1. match
```
    1. __match_args__ - задает естественный порядок атрибутов класса
```

### 2. Вызываемые объекты:
```
    1. __call__  - позволяет любому экземпляру класса вести себя как обычная функция
    2. __await__ - возвращает итератор, превращая класс в корутину
```

### 3. Доступ к атрибутам объекта
```
    1. __getattribute__(self, item) – автоматически вызывается при получении свойства класса с именем item(x.y);
    2. __getattr__(self, item) - вызывается при получении несуществующего свойства, обращении к атрибуту объекта через x.y
    3. __setattr__(self, key, value) - Например: x.y = 1
    4. __delattr__(self, item) - del x.y
    5. __dir__ - вызывается методом dir(...) и выводит список доступных атрибутов объекта.
```

### 4. Протокол дескрипторов
```
    1. descr.__get__(self, obj, type=None) -> value 
    2. descr.__set__(self, obj, value) -> None   
    3. descr.__delete__(self, obj) -> None 
    
    4. __set_name__
```

### 5. Контекстные менеджеры
```
    1. __enter__(self) — определяет начало блока контекстного менеджера, вызванного с помощью with
    2. __exit__(self, exc_type, exc_value, traceback) — определяет конец блока контекстного менеджера. 
    
    3. __aenter__(self) — аналогично __enter__, только функция возвращает корутину (результат которой можно получить с 
                          помощью await)
    4. __aexit__(self, exc_type, exc_value, traceback) — аналогично __exit__, только функция возвращает корутину 
                                                         (результат которой можно получить с помощью await)
```

### 6. Числовые методы
##### 6.1. Унарные операторы
```
    1. __neg__(self)    — определяет поведение для отрицания (-a)
    2. __pos__(self)    — определяет поведение для унарного плюса (+a)
    3. __abs__(self)    — определяет поведение для встроенной функции abs(...)
    4. __invert__(self) — определяет поведение для инвертирования оператором ~
```

##### 6.2. Обычные арифметические операторы
```
    1. __add__(self, other) — сложение, оператор +
    2. __sub__(self, other) — вычитание, оператор -
    3. __mul__(self, other) — умножение, оператор *
    4. __matmul__(self, other) — умножение матриц, оператор @
    5. __truediv__(self, other) — деление, оператор /
    6. __floordiv__(self, other) — целочисленное деление, оператор //
    7. __mod__(self, other) — остаток от деления, оператор %
    8. __divmod__(self, other) — деление с остатком, определяет поведение для встроенной функции divmod(...)
    9. __pow__(self, other[, modulo]) — возведение в степень, оператор **
    10. __lshift__(self, other) — двоичный сдвиг влево, оператор <<
    11. __rshift__(self, other) — двоичный сдвиг вправо, оператор >>
    12. __and__(self, other) — двоичное И, оператор &
    13. __xor__(self, other) — исключающее ИЛИ, оператор ^
    14. __or__(self, other) — двоичное ИЛИ, оператор |
```

##### 6.3. Отражённые арифметические операторы
```
    1. __radd__(self, other) — сложение, оператор +
    2. __rsub__(self, other) — вычитание, оператор -
    3. __rmul__(self, other) — умножение, оператор *
    4. __rmatmul__(self, other) — умножение матриц, оператор @
    5. __rtruediv__(self, other) - Отражённое правильное деление. Заметьте, что работает только когда используется from __future__ import division.
    6. __rdiv__(self, other) - Отражённое деление, оператор /.
    7. __rfloordiv__(self, other) — целочисленное деление, оператор //
    8. __rmod__(self, other) — остаток от деления, оператор %
    9. __rdivmod__(self, other) — деление с остатком
    10. __rpow__(self, other[, modulo]) — возведение в степень, оператор **
    11. __rlshift__(self, other) — двоичный сдвиг влево, оператор <<
    12. __rrshift__(self, other) — двоичный сдвиг вправо, оператор >>
    13. __rand__(self, other) — двоичное И, оператор &
    14. __rxor__(self, other) — исключающее ИЛИ, оператор ^
    15. __ror__(self, other) — двоичное ИЛИ, оператор |
```

##### 6.4.Составное присваивание
```
    1. __iadd__(self, other) — сложение с присваиванием, оператор +=
    2. __isub__(self, other) — вычитание с присваиванием, оператор -=
    3. __imul__(self, other) — умножение с присваиванием, оператор *=
    4. __imatmul__(self, other) — умножение матриц с присваиванием, оператор @=
    5. __itruediv__(self, other) - Правильное деление с присваиванием. Заметьте, что работает только если используется from __future__ import division.
    6. __idiv__(self, other) - Деление с присваиванием, оператор /=.
    7. __ifloordiv__(self, other) — целочисленное деление с присваиванием, оператор //=
    8. __imod__(self, other) — остаток от деления с присваиванием, оператор %=
    9. __ipow__(self, other[, modulo]) — возведение в степень с присваиванием, оператор **=
    10. __ilshift__(self, other) — двоичный сдвиг влево с присваиванием, оператор <<=
    11. __irshift__(self, other) — двоичный сдвиг вправо с присваиванием, оператор >>=
    12. __iand__(self, other) — двоичное И с присваиванием, оператор &=
    13. __ixor__(self, other) — исключающее ИЛИ с присваиванием, оператор ^=
    14. __ior__(self, other) — двоичное ИЛИ с присваиванием, оператор |=
```

### 7. Создание последовательностей
```
    1. __len__(self) — вызывается методом len(...) и возвращает количество элементов в последовательности.    
    2. __getitem__(self, key) — вызывается при обращении к элементу в последовательности по его ключу (индексу). 
                                Метод должен выбрасывать исключение TypeError, если используется некорректный тип ключа, 
                                KeyError, если данному ключу не соответствует ни один элемент в последовательности. Например:
                             
    >>> list_object = [1, 2, 3, 4, 5]
    >>> print(list_object[0])
    ... 1    
    < --------------------------------------------------------------------------------------------------------------- >
    3. __setitem__(self, key, value) — вызывается при присваивании какого-либо значения элементу в последовательности. 
                                       Также может выбрасывать исключения TypeError и KeyError. Например:                   
    >>> list_object = [1, 2, 3, 4, 5]
    >>> list_object[0] = 78
    >>> print(list_object)
    ... [78, 2, 3, 4, 5]   
    < --------------------------------------------------------------------------------------------------------------- >                                    
    4. __delitem__(self, key) — вызывается при удалении значения в последовательности по его индексу (ключу) с 
                                помощью синтаксиса ключевого слова del.
    5. __missing__(self, key) — вызывается в случаях, когда значения в последовательности не существует.
    6. __iter__(self) — вызывается методом iter(...) и возвращает итератор последовательности                
    7. __aiter__(self) - должен возвращать асинхронный итерируемый объект и семантически похож на метод __iter__() простого итератора.
    8. __next__(self) - Метод должен возвращать следующий элемент из контейнера, а если элементы в последовательности 
                        закончились, то метод container.__next__() должен бросить исключение StopIteration.
                        Как только метод container.__next__() итератора вызывает StopIteration, он должен продолжать 
                        вызывать его и при последующих вызовах. Реализации, которые не подчиняются этому свойству, 
                        считаются нарушенными.
    9. __anext__(self) - Асинхронный метод object.__anext__() семантически похож на метод __next__() простого итератора, с 
                         той лишь разницей, что ондолжен возвращать следующее значение итератора в результате awaitable.
    10. __reversed__(self) — вызывается методом reversed(...) и аналогично методу __iter__ возвращает тот же итератор, 
                            только в обратном порядке.
    11. __contains__(self, item) — вызывается при проверке принадлежности элемента к последовательности с помощью in или not in.          
    
    < --------------------------------------------------------------------------------------------------------------- >  
    
    12. object.__length_hint__() вызывается operator.length_hint() - Метод object.__length_hint__() вызывается для 
                    реализации operator.length_hint(). Должен возвращать оценочную длину объекта (которая может быть 
                    больше или меньше фактической длины). Длина должна быть целым числом >= 0. Возвращаемое значение 
                    также может быть NotImplemented, что обрабатывается так же, как если бы метод __length_hint__ 
                    вообще не существовал. Этот метод является чисто оптимизационным и никогда не требуется для 
                    корректности.                                                          
```

### 8. Представление своих классов
```
    Любой объект может содержать дополнительную информацию, полезную при отладке или приведении типов. Например:
    
    1. __str__(self) — вызывается при вызове функции str(...), возвращает строковый объект.
    
    2. __repr__(self) — информационная строка об объекте. 
                        Выводится при вызове функции repr(...) или в момент отладки. 
                        Для последнего этот метод и предназначен.
                        
    3. __unicode__ — полностью исчез в версии Python 3, вместо него используются отдельные методы __str__и __bytes__
    
    4. __format__(self, format_spec) — вызывается при вызове функции format(...) и 
                                       используется для форматировании строки с использованием строковых литералов.    
                                                             
    5. __hash__(self) - Определяет поведение функции hash(), вызыванной для экземпляра вашего класса. 
                        Метод должен возвращать целочисленное значение, которое будет использоваться для быстрого 
                        сравнения ключей в словарях. Заметьте, что в таком случае обычно нужно определять и __eq__ тоже. 
                        Руководствуйтесь следующим правилом: a == b подразумевает hash(a) == hash(b).    
                     
    6. __nonzero__ — переименован в __bool__   
       __nonzero__(self) - Определяет поведение функции bool(), вызванной для экземпляра вашего класса. 
                           Должна вернуть True или False, в зависимости от того, когда вы считаете экземпляр 
                           соответствующим True или False.                                          
    
    7. __dir__(self) - Определяет поведение функции dir(), вызванной на экземпляре вашего класса. 
                       Этот метод должен возвращать пользователю список атрибутов. 
                       Обычно, определение __dir__ не требуется, но может быть жизненно важно для интерактивного 
                       использования вашего класса, если вы переопределили __getattr__ или __getattribute__ 
                       (с которыми вы встретитесь в следующей части), или каким-либо другим образом динамически 
                       создаёте атрибуты.

    8. __bytes__(self) — аналогично __str__(self), только возвращается набор байт.
    
    9. __sizeof__(self) - Определяет поведение функции sys.getsizeof(), вызыванной на экземпляре вашего класса. 
                          Метод должен вернуть размер вашего объекта в байтах. Он главным образом полезен для классов, 
                          определённых в расширениях на C, но всё-равно полезно о нём знать.   
```

### 9. Преобразования типов
```   
    1. __complex__(self) — преобразование типа в комплексное число
    2. __int__(self) — преобразование типа к int
    3. __float__(self) — преобразование типа к float
    4. __index__(self) — Преобразование типа к int, когда объект используется в срезах (выражения вида [start:stop:step]). 
                         Если вы определяете свой числовый тип, который может использоваться как индекс списка, 
                         вы должны определить __index__.
    5. __round__(self[, ndigits]) — округление числа с помощью функции round(...)
    6. __trunc__(self) — Вызывается при math.trunc(self). Должен вернуть своё значение, обрезанное до целочисленного типа (обычно long).
    7. __floor__(self) — вызывается методом math.floor(...)
    8. __ceil__(self) — вызывается методом math.ceil(...)
    9. __long__(self) - Преобразование типа в long.
    10. __oct__(self) - Преобразование типа в восьмеричное число.
    11. __hex__(self) - Преобразование типа в шестнадцатиричное число.
    12. __coerce__(self, other) - Метод для реализации арифметики с операндами разных типов. 
                                 __coerce__ должен вернуть None если преобразование типов невозможно. 
                                 Если преобразование возможно, он должен вернуть пару (кортеж из 2-х элементов) из self 
                                 и other, преобразованные к одному типу.
```

### 10. Создание и удаление объектов
```
    1. __new__(cls[, ...]) — метод создания типа класса. 
                             Он принимает первым аргументом тип класса, в котором он вызывается, и, как правило, 
                             возвращает этот же тип. В основном используется, чтобы настраивать создание экземпляра 
                             класса тех объектов, которые наследуются от неизменяемых типов (например, int, str, или tuple).
    
    2. __init__(self[, ...]) — конструктор класса. Используется при определении объектов.
    
    3. __init_subclass__(cls) — позволяет переопределить создание подклассов объекта. Например, добавлять дополнительные атрибуты
    
    4. __del__(self) — деструктор класса. 
                       Вызывается автоматически сборщиком мусора, практически никогда не используется, за исключением, 
                       когда пользователя необходимо предупредить о незакрытых дескрипторах.


    Пример паттерна Singleton (учебный)

    def __new__(cls, *args, **kwargs):
        if cls.__instance is None:
            cls.__instance = super().__new__(cls)
 
        return cls.__instance
```
