#### Сергей Тарасов. СУБД для программиста. БД изнутри.  
```
    1. Первичный сбор и индексация информации, чью структуру нет возможности классифицировать настолько полно, 
    чтобы спроектировать схему БД.
    
    2. Класс задач, для которых нет оптимальных решений с помощью реляционных БД.
    
    В первую очередь это сущности со сложной и многоуровневой иерархией вложения.
    Если хранить такие сущности в реляционных БД, то каждый тип должен быть в нормализованном виде отражен 
    на десятки таблиц. В результате обработка таких данных будет сопровождаться проблемами с производительностью.
    Код запросов будет сложен и труден в сопровождении.
    В таком случае лучшим решением будет использование СУБД поддерживающих многомерные и сетевые модели данных.
```
